Managing users and groups
 
Overview
Build up a heirarcy of groups to represent your institution. Add students, teachers, reporters to the groups. All users have details such as names, passwords, expiry dates that you can manage. You can move, delete and add users.
   
Details

Users have names and IDs that can be used for login and for displaying on progress records.
Passwords and used for login.
Expiry dates are used to block access to users after a certain date.
Email is used by Results Manager to send out information or alert messages, and to lets students use the "I forgot my password" feature during login.
   
Types of users
Each account has an admin user that you can't move out of the top level group. Admin can do anything.
Teachers can do most things within their group, but they don't see any users or groups outside. Teachers can add new groups and add and import users. Teachers can get reports. Teachers can create new content in Author Plus and Clarity Course Builder, if you have those programs. Teachers can't see other teacher's details.
Reporters can only generate reports. They can't do anything else.
   
   
Reports
 
Overview
You can generate reports for printing or viewing on screen.
   
Building the report

Right click on a group, user or content item. You then see a report building screen where you choose what information you want to see. If you are making a report for a user or group, the main choice is what content you want in the report - and what level of detail you want. You can also filter out records based on dates or scores.

See report types for more details.
Screen report
When you generate the report it will create a new page in your browser with the report. This report is a grid that you can search, order and group as you like.
   
Print report
This is the same report, but with simple formatting that will work on the printer. It is NOT a copy of the grid.
   
   
Available content
 
Overview
You might not want all students to have all the content available to them at the same time.
   
Hiding content

Click on the group that you want to hide content from. You will then see ticks appear in the content box - these ticks show which content is available to students in this group. In the first place everything is available. So to hide an item of content you click on the tick which will disappear.

So, if I want students in the group "Year 4 EFT" to only work on Tense Buster Elementary Am, is, are and not see the other units, I click group Year 4 EFT, then expand Tense Buster Elementary. I click on Elementary which hides everything, then click on Am, is, are which makes that unit available. Click to watch...
   
   
Search
 
Overview
You need to find a particular user, or a set of users from the list.
   
Search fields

Right-click on the group you want to search in. You can now type in the details you want to search for.

  • Name, Learner ID, Email, Country. You can search for any part of these fields. Searching for 'Bob' will find 'Bob King', 'Bobby Lee' and 'Eileen Perbobs'.
  • Type of user and expiry date are based on a selected date or dropdown list.

If you try to delete a group from the search screen you will be warned that you might be deleting hidden users. You should always clear the search before you delete a group. But you can safely select all the users you see and delete them.
   
Clearing a search The results of the search are shown in the same user and group list. But only those users who match the search are shown. The list is given a coloured background to show that this is not all users. You can search again to get rid of even more users, or you can click the Clear Search button to go back to your full list.

Importing users
 
Overview
If you have a spreadsheet or document that contains the details of all the users you want to add to results manger you can add all of them at once using Import users. You can also import from an XML document if you want (to be implemented later).
   
From Excel
Start at the Users, Reports, Content screen. Click on the group that you want to add the new users to. If you want to add a new group, click on the top group in your list. Right click and choose Import users. Flip to your spreadsheet and select the data you want to import. This must include a row of headers, –see later for more details and an example. Copy the data to the clipboard (Ctrl-C). Flip back to RM and click in the empty box (number 2) and paste (Ctrl-V). Click the Start import button. RM will now add your users. If there are any problems it will show you a message next to the users who can't be imported. If you do have any failed imports, you will have to change their details in the spreadsheet and repeat the process.
From Word
You can also import in exactly the same way from a table in Word - in fact from any source that creates a tab-delimited list.
   
What can I import?
Your spreadsheet must at least have a 'Name' or 'ID' column.
  • Name
  • ID
You can also have columns for the following information:
  • Password
  • Email
  • Country
  • Usertype - special settings 'Learner/Student', 'Teacher', 'Reporter', 'Author'. If you leave this blank it is assumed to be a student.
  • Expiry date – must be in YYYY/MM/DD format. Use Excel to change your data format if necessary. If you leave this blank the user's account will not expire.
  • City
  • Country
  • Group (see below)
   
Can I add students in different groups at once?
Yes. You can add a Group column to your spreadsheet. If this group matches an existing one, the student will be added to it. If there is no matching group, a new one will be made. Group names are only matched within the group that you clicked on to start the import. So you can have a group called 'Class 1' in both the 'Year 1' and 'Year 2' groups and import will put students into Class 1 based on whether you are importing into Year 1 or Year 2.
   
Can I create groups that don't already exist?
Yes. If the group doesn't exist it will be created. You can also have a heirarchy of groups listed in the group column. So /Year 1/Class A/Elementary will add this user to Elementary in Class A in Year 1.
   
What about users who are in the spreadsheet and already in Results Manager?
Next to the Import Users button, there is an option that lets you move or copy existing users. So if a duplicate name or id already exists in Results Manager, that user will be moved or copied to the new group that you type. This means that if your users change groups at year end, you can import them all and their existing records will move to the correct group for the next year.
   
Can I put a student into two different groups?
Yes. Next to the Import Users button, there is an option that lets you copy existing users. So if the user already exists in one group, importing them will copy them to a second group. You can have the same student listed many times in the spreadsheet to do this in one go.
   
Does the order of the columns matter?
No. It is only important to get the names of the columns correct and to make sure you include the column names in the data you copy and paste.
   
Are the names of the columns important?
Yes. There are some variations, listed below, but you must basically use a preset name for each column. Columns names are not case-sensitive, so EMail is the same as email.
  • name, username, user name, user_name, student name
  • id, student id, id_no, id_number, learner id
  • usertype, type
  • email, e-mail
  • expiry date, expiry, expirydate, expiry_date
  • city, region
  • group, group name, group_name, groupname
Can I undo an import?
If you import students you can immediately delete them with no impact. This doesn't do anything to your licence.
